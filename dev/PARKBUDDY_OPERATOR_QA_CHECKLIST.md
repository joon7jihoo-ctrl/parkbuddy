# ParkBuddy Operator QA Checklist

## 목적

ParkBuddy 운영 배포 전후에 운영자가 직접 확인해야 할 핵심 흐름을 정리한다. 이 문서는 새 기능 개발보다 **현재 기능이 끊기지 않고 자연스럽게 이어지는지**를 확인하기 위한 기준이다.

## 공통 확인

```powershell
cd C:\parkbuddy
npm run verify
```

기대 결과:

```text
Security smoke test passed.
ESLint passed.
TypeScript passed.
```

배포 후 확인 주소:

```text
https://parkbuddy-five.vercel.app
```

## 1. 로그인 / 초대 코드

확인 화면:

```text
/login
/login?invite=1
```

확인 항목:

- 카카오 로그인 버튼이 보이는지
- 초대 코드로 시작하기 경로가 보이는지
- 카카오 로그인 후 `/auth/callback`으로 정상 복귀하는지
- 모바일 데이터망에서 카카오가 실패하더라도 초대 코드 화면은 접근 가능한지
- 초대 코드 화면의 입력 항목이 이름, 연락처, 초대 코드 중심으로 단순한지

Supabase 확인:

- Site URL: `https://parkbuddy-five.vercel.app`
- Redirect URLs:
  - `https://parkbuddy-five.vercel.app/auth/callback`
  - `https://parkbuddy-five.vercel.app/**`
  - `http://localhost:3000/auth/callback`
  - `http://localhost:3000/**`
- Kakao Developers Redirect URI:
  - `https://jefbokflyuywzjotwmhn.supabase.co/auth/v1/callback`
- Anonymous sign-ins 활성화 여부

## 2. 홈 / 게시판

확인 화면:

```text
/
/board
/board/new
```

확인 항목:

- 홈 화면에 불필요한 설명 문구가 없는지
- 주요 메뉴 이동이 제목만으로 자연스러운지
- “최근 공지”처럼 게시판과 같은 기능을 다른 이름으로 부르는 표현이 남아 있지 않은지
- 게시글 카드를 눌렀을 때 상세로 이동하는지
- “자세히 보기” 같은 중복 문구가 보이지 않는지
- 글쓰기 유형이 “공지”, “자유글”처럼 한글 중심으로 보이는지

## 3. 회원 관리 / 초대 코드 공유

확인 화면:

```text
/admin/members
/members
```

확인 항목:

- 회원 검색이 정상 동작하는지
- 불필요한 설명 없이도 회원 관리 흐름이 이해되는지
- 미연결 회원의 초대 코드 발급/재발급이 가능한지
- 초대 코드 복사와 전체 안내문 복사가 정상 동작하는지
- 회원 목록 카드가 모바일에서 너무 길지 않은지

## 4. 일정 / 참석 투표 / 라운딩 생성

확인 화면:

```text
/schedule
```

확인 항목:

- 일정 카드에서 참석/불참 투표가 정상 동작하는지
- 참석자 목록 모달이 중앙 정렬로 자연스럽게 뜨는지
- 운영자에게만 라운딩 생성 영역이 보이는지
- 참석자 기준 라운딩 생성 전 참석/불참/미선택 수가 맞는지
- 참석자가 0명이면 생성 버튼 대신 참석자 필요 안내가 보이는지
- 이미 생성된 일정은 “생성된 라운딩 보기”가 보이는지

## 5. 라운딩 목록 / 연결 일정

확인 화면:

```text
/admin/rounds
```

확인 항목:

- 모바일에서 상태 필터가 compact하게 보이는지
- 라운딩 카드가 Dense UX로 유지되는지
- 경기 방식이 “개인전 신페리오”처럼 한글로 표시되는지
- 참가자 / 조 편성 / 스코어 / 결과 버튼이 모바일에서 빠르게 보이는지
- 일정에서 생성된 라운딩은 연결 일정 정보가 보이는지
- 수동 생성 라운딩은 연결 일정 정보가 보이지 않는지

## 6. 참가자 관리

확인 화면:

```text
/admin/rounds/[id]/participants
```

확인 항목:

- 검색 카드가 상단 compact 영역으로 보이는지
- 선택 수가 검색 카드, 안내 문구, 저장 버튼에 함께 반영되는지
- 선택만 보기 / 전체 보기 토글이 정상 동작하는지
- 표시 전체 선택 / 표시 해제가 현재 검색 결과 기준으로 동작하는지
- 참가자 저장 후 다시 들어왔을 때 유지되는지

## 7. 조 편성

확인 화면:

```text
/admin/rounds/[id]/pairings
```

확인 항목:

- “수동 조 편성” 대신 “조 편성”으로 보이는지
- 경기 방식 카드가 콤보박스 2개와 설명 1개만으로 구성되어 있는지
- 경기 형태 / 점수 계산 방식이 한글로 보이는지
- 참가자 3~4명은 1조 편성이 정상임을 인지할 수 있는지
- 참가자 6명 이상은 여러 조로 자동 분산되는지
- 경기 형태를 바꾸면 조 편성 초안이 다시 계산되는지
- 회원별 조 선택 콤보박스로 임의 변경 가능한지
- 저장 후 다시 들어왔을 때 조 편성이 유지되는지

## 8. 스코어 입력

확인 화면:

```text
/admin/rounds/[id]/scores
```

확인 항목:

- 스코어 입력 카드가 모바일에서 너무 길지 않은지
- 완료된 회원은 compact하게 접히는지
- 미입력만 보기 필터가 정상 동작하는지
- 점수 입력 중 카드가 갑자기 사라지지 않는지
- 저장 버튼에 완료 수가 자연스럽게 보이는지
- 저장 후 결과 화면으로 이동하거나 결과 확인이 가능한지

## 9. 결과 / 인쇄 / 사용자 기록

확인 화면:

```text
/admin/rounds/[id]/results
/admin/rounds/[id]/results/print
/scores
/scores/[roundId]
/mypage
```

확인 항목:

- 결과 화면에서 “상위 3명”이 한글로 보이는지
- 계산 기준은 기본 노출이 아니라 접이식으로 보이는지
- 인쇄 화면이 깨지지 않는지
- `/scores`와 `/mypage` 최근 기록이 같은 기록 기준으로 보이는지
- `round_scores`에 입력한 기록이 사용자 화면에도 반영되는지

## 배포 전 최종 순서

1. `npm run verify`
2. 로컬 주요 화면 확인
3. 커밋/푸시
4. Vercel 배포 완료 확인
5. 모바일 Chrome에서 운영자 계정으로 핵심 흐름 확인
6. 이상 화면 캡처 후 다음 수정 범위 결정

# ParkBuddy Next Development Flow

## 목적

현재 ParkBuddy는 기능이 빠르게 늘어난 상태다. 다음 개발부터는 새 기능을 바로 추가하기보다, 안정화된 흐름을 유지하면서 단계적으로 개선한다.

## 현재 안정화된 핵심 흐름

1. 로그인
   - 카카오 로그인
   - 초대 코드 기반 첫 접속
   - 운영자용 초대 코드 공유

2. 일정
   - 일정 생성/조회
   - 참석/불참 투표
   - 참석자 기준 라운딩 생성
   - 생성 전 확인 UX

3. 라운딩 관리
   - 라운딩 목록
   - 연결 일정 정보 표시
   - 참가자 관리
   - 조 편성
   - 스코어 입력
   - 결과/인쇄

4. 사용자 화면
   - 마이페이지
   - 스코어 목록/상세
   - 회원 목록
   - 게시판

## 다음 개발 원칙

### 1. 화면을 먼저 확인한다

코드를 수정하기 전, 반드시 현재 화면을 먼저 확인한다.

- 모바일 폭
- 태블릿 폭
- 데스크톱 폭
- 운영자 계정
- 일반 회원 계정

### 2. 한 번에 여러 핵심 흐름을 건드리지 않는다

예를 들어 조 편성 화면을 수정하는 작업에서 스코어 입력, 회원 검색, 일정 투표 로직을 함께 건드리지 않는다.

### 3. 안정화된 흐름은 보호한다

특히 아래 기능은 이미 여러 번 안정화되었으므로 요청 없이는 수정하지 않는다.

- 회원 검색
- 스코어 입력 자동 접힘/미입력 필터
- 일정 참석/불참 투표
- 카카오 OAuth redirect
- 초대 코드 로그인
- 라운딩 목록 Dense UX

### 4. 문구는 최소화한다

화면 문구 기준:

- 제목과 버튼만으로 이해되면 설명을 넣지 않는다.
- 클릭하면 알 수 있는 행동은 “자세히 보기”처럼 반복하지 않는다.
- 같은 화면/기능은 같은 이름으로 부른다.
- `ParkBuddy` 외 영어는 사용자 화면에서 피한다.

### 5. 데이터 기준을 통일한다

스코어/통계는 `round_scores`를 공식 기준으로 사용한다.

새 화면에서 스코어 기록을 보여줄 때는 먼저 아래를 확인한다.

- `round_scores` 기반인지
- My Page / Scores / Results와 같은 기준인지
- 예전 `hole_scores` 또는 `round_players` 기준이 섞이지 않았는지

## 다음 추천 작업 순서

### 1순위. 운영자 QA 기반 버그 정리

목표:

- 지금까지 만든 기능을 모바일에서 실제로 눌러보고 깨지는 부분을 먼저 수정한다.

대상:

- `/login`
- `/schedule`
- `/admin/rounds`
- `/admin/rounds/[id]/participants`
- `/admin/rounds/[id]/pairings`
- `/admin/rounds/[id]/scores`
- `/admin/rounds/[id]/results`

산출물:

- 화면별 캡처
- 이상 여부 기록
- 버그 수정 패치

### 2순위. 조 편성 알고리즘 검증/고도화

현재 상태:

- 평균타수와 참여횟수 기반 실력 균등 편성 초안 도입
- 팀 데이터가 부족하면 실력 균등으로 fallback

다음 확인:

- 실제 회원 데이터에 평균타수/참여횟수가 충분히 있는지
- 참가자 6명 이상에서 조가 분산되는지
- 경기 형태 변경 시 재계산되는지
- 팀/소속 데이터가 필요한지

추가 설계 후보:

- `members.team_no` 또는 별도 팀/소속 필드 공식화
- 청백전 팀 배정 UI
- 포섬/포볼 2:2 팀 혼합 강제 로직

### 3순위. 인증 고도화 재검토

현재 상태:

- 카카오 로그인 유지
- 초대 코드 첫 접속 추가
- 패스키는 장기 방향으로 문서화

다음 확인:

- 커스텀 도메인 사용 여부
- Supabase passkey experimental 상태 변화
- Kakao SSO 사용자와 passkey 연결 가능성

추천:

- 운영 도메인이 확정되기 전까지 passkey 실제 구현은 보류
- 초대 코드/카카오 흐름의 실사용 안정성을 먼저 확보

### 4순위. 남은 화면 Dense UX 정리

후보:

- 삭제/복구 라운드 화면
- 월간 라운딩 보기
- 운영 로그
- 회원 상세/수정
- 게시글 상세/댓글 영역

기준:

- 모바일에서 스크롤을 줄인다.
- 카드 안 설명을 줄인다.
- 버튼은 핵심 행동만 남긴다.

### 5순위. 결과 공유/인쇄 품질 개선

후보:

- 결과 공유 이미지/인쇄 레이아웃
- 스코어표 가독성
- 상위 3명 카드 정리
- 계산 기준 표시 방식 개선

# 조 편성 저장 인원 오류 수정 보고서

## 작업 배경

조 편성 화면에서는 참가자 8명을 정상적으로 읽고 있었지만, 저장 시에는 서버 액션이 참가자 목록을 0명으로 판단하여 `조 편성을 위해 최소 3명 이상의 참가자가 필요합니다` 오류가 발생했다.

## 원인

조 편성 폼은 `assignments` hidden input에 JSON 형태로 참가자별 조 정보를 보내고 있었다.

하지만 서버 액션은 `memberId`, `groupNo:{memberId}` 형식만 읽고 있어서 실제 저장 요청의 참가자 목록을 비어 있는 배열로 처리했다.

## 수정 내용

- 조 편성 저장 액션이 `assignments` JSON을 우선 읽도록 수정
- 기존 `memberId`, `groupNo:{memberId}` 방식도 예비 경로로 유지
- 조 번호를 화면의 0 기준 값에서 DB 저장용 1 기준 값으로 변환
- 경기 형태와 점수 계산 방식 조합 검증 유지
- 저장 실패 시 로그에 assignments 개수를 함께 남기도록 보강

## 변경 파일

- `src/app/(app)/admin/rounds/[id]/pairings/actions.ts`

## 확인 항목

1. 참가자 8명을 선택한다.
2. 참가자 저장 후 조 편성 화면으로 이동한다.
3. 추천 조 편성이 4명 / 4명으로 표시되는지 확인한다.
4. 청백전 + 스트로크 플레이를 선택하고 저장한다.
5. 최소 3명 오류가 사라졌는지 확인한다.

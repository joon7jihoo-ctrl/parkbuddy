# ParkBuddy 스코어 입력 태블릿/데스크탑 입력창 배치 수정

## 작업 일시
- 2026-06-15

## 배경
스코어 입력 화면에서 태블릿/데스크탑 폭에서도 `스테이블포드 포인트` 입력창이 회원명 아래쪽으로 내려가 보이는 문제가 있었다.
운영 화면에서는 회원명, 총 타수, 스테이블포드 포인트가 한 줄에 나란히 보이는 편이 입력 속도와 가독성에 유리하다.

## 변경 범위
- `src/components/admin/round-score-input-form.tsx`

## 변경 내용
- 모바일 기본 배치는 유지했다.
- `sm` 이상 화면부터 참가자명 / 총 타수 / 스테이블포드 포인트를 3열 그리드로 배치했다.
- 입력 label에 `min-w-0`을 추가해 그리드 내부에서 입력창이 불필요하게 줄바꿈되거나 아래로 밀리지 않도록 했다.

## 검증
- `npm run verify` 통과
  - security smoke test 통과
  - eslint 통과
  - typecheck 통과

## 화면 확인 포인트
1. 모바일 폭에서는 기존처럼 입력창이 보기 좋게 세로/2열 흐름으로 표시되는지 확인한다.
2. 태블릿/데스크탑 폭에서는 한 참가자 카드 안에서 다음 순서로 한 줄 배치되는지 확인한다.
   - 회원명/핸디캡/입력상태
   - 총 타수
   - 스테이블포드 포인트
3. `미입력만 보기` 상태에서 60처럼 두 자리 숫자 입력 시 카드가 입력 중에는 사라지지 않고 저장값도 유지되는지 다시 확인한다.

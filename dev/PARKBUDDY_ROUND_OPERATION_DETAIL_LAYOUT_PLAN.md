# ParkBuddy 라운드 운영 상세 화면 레이아웃 개선 계획

## 작업명

Polish round operation detail screens

## 목적

라운드 목록, 월별 일정, 상태별 보기, 삭제된 라운드 화면에 적용한 모바일 우선 카드 레이아웃을 라운드 운영 상세 화면까지 확장한다.

## 적용 화면

- /admin/rounds/[id]/participants
- /admin/rounds/[id]/pairings
- /admin/rounds/[id]/scores
- /admin/rounds/[id]/results
- /admin/rounds/[id]/edit

## 레이아웃 원칙

1. 데스크탑에서는 본문 최대 폭을 넓혀 정보가 왼쪽에 눌리지 않게 한다.
2. 모바일에서는 핵심 정보 요약을 2~3칸 카드로 압축한다.
3. 버튼은 화면 크기에 맞춰 2열, 가로 dock, 우측 작업 영역으로 자연스럽게 이동한다.
4. 회원/스코어/결과 목록은 무조건 세로 나열하지 않고 카드 또는 2열 그리드로 압축한다.
5. 저장 버튼은 모바일에서 하단 네비게이션과 겹치지 않는 선에서 찾기 쉽게 배치한다.

## 검증 기준

- npm run verify 통과
- 모바일에서 각 상세 화면의 첫 화면 정보량이 증가해야 한다.
- 데스크탑에서 라운드 정보가 좁게 눌리거나 세로로 깨지지 않아야 한다.
- 참가자 선택 화면의 회원 카드가 모바일/태블릿/데스크탑에서 자연스럽게 배치되어야 한다.
- 스코어 입력 화면에서 한 참가자의 입력 영역이 지나치게 길게 늘어나지 않아야 한다.
- 결과 화면의 순위 카드가 모바일에서도 한눈에 읽혀야 한다.

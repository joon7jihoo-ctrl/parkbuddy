## 구현 기능

운영진 회원 관리 화면에 활성 회원/비활성 회원 보기 탭을 추가했다.

비활성 회원 보기에서는 비활성 처리된 회원을 확인하고 복구할 수 있다.

복구된 회원은 다시 활성 회원 목록에 표시된다.

## DB 보안 함수

다음 RPC를 추가했다.

```txt
public.admin_restore_member(...)
```

이 함수는 DB 내부에서 다음을 검증한다.

```txt
1. 로그인 여부
2. 운영진 권한 여부
3. 같은 동호회 회원인지 여부
4. 복구 대상이 inactive 상태인지 여부
5. 복구 시 같은 동호회 내 활성 회원과 연락처가 중복되는지 여부
```

## 보안 평가

복구 버튼은 운영진 화면에만 보이지만, 최종 권한 검증은 Supabase RPC에서 다시 수행한다.

비활성 회원 복구 시 연결 코드는 자동으로 새로 만들지 않는다.

계정 미연결 회원이 복구된 경우 운영진이 필요할 때 별도로 연결 코드를 재발급해야 한다.

이 방식은 불필요한 연결 코드 발급을 줄이고, 로그와 권한 관리를 명확하게 유지한다.

## 완료 기준

```txt
1. Supabase SQL Editor에서 0010_admin_restore_member.sql 실행
2. npm run verify 통과
3. /admin/members 화면에 활성 회원/비활성 회원 탭 표시
4. 비활성 회원 탭에서 비활성 회원 확인 가능
5. 복구 버튼 클릭 시 확인창 표시
6. 복구 후 활성 회원 목록에 다시 표시
7. 최근 관리자 작업에 회원 복구 로그 표시
```

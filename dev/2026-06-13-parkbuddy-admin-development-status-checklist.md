# ParkBuddy 운영 개발 현황 및 다음 단계 점검표

## 작성일

2026-06-13

## 목적

지금까지 진행한 ParkBuddy 관리자 기능을 한눈에 확인하고, 다음 개발 단계에서 같은 기능을 중복 구현하거나 기존 흐름을 깨지 않도록 기준 문서로 사용한다.

- 카카오 로그인 기반 인증 흐름

- 회원 검색
- 회원 정보 수정
- 회원 비활성화 및 복구

- claim code 복사 버튼
- 관리자 작업 로그 화면
- 관리자 작업 로그 한글 표시

### 라운드 관리

- 라운드 생성
- 라운드 목록

- 라운드 상태 변경

  - 취소

- 라운드 복제

- 보정 타수 기반 결과 표시

## 현재 주의해야 할 점

### Supabase SQL 실행이 필요한 기능

아래 기능은 로컬 파일 생성만으로 끝나지 않고, Supabase SQL Editor에서 migration SQL을 실행해야 한다.

- 라운드 상태 관리
- 라운드 복제
- 스코어 입력 기반
- 라운드 관련 RPC 추가 기능

### 관리자 로그 관련 주의점

관리자 로그 함수의 실제 시그니처와 기존 호출 방식이 일부 맞지 않았던 이력이 있다.

따라서 다음부터 로그를 추가할 때는 먼저 현재 `admin_log_action` 함수 구조를 기준으로 확인한 뒤 붙인다.

### PowerShell 스크립트 작성 원칙

앞으로 실행 파일은 기본적으로 `.ps1`로 제공한다.

단, 긴 코드 문자열을 PowerShell에 직접 넣지 않는다.

권장 방식:

- PS1은 짧게 유지
- 실제 파일 내용은 Base64로 복원
- 복잡한 수정 로직은 임시 Node 파일을 만들어 실행
- 작업 완료 후 임시 파일 삭제
- 마지막에 `npm run verify` 실행

## 다음 추천 개발 단계

### 1순위: 라운드 복제 안정화 최종 점검

라운드 복제는 최근 여러 DB 컬럼/함수 차이로 오류가 발생했으므로, 실제 동작 확인이 필요하다.

확인 항목:

- 복제 버튼 클릭
- 새 라운드 생성
- 새 라운드 상태가 예정인지 확인
- 참가자/조/스코어가 복사되지 않는지 확인
- 참가자 선택 화면으로 이동하는지 확인

### 2순위: 스코어 입력 UX 개선

현재 스코어 입력 기반은 있으므로 다음 개선이 적합하다.

- 저장 후 결과 보기 버튼 표시
- 입력된 스코어가 있는 참가자는 입력 완료 표시
- 미입력 참가자 수 표시
- 모바일 입력 편의성 개선

### 3순위: 결과 계산 고도화

신페리오와 매치 플레이를 제대로 계산하려면 홀별 스코어 입력 기반이 필요하다.

권장 순서:

1. 홀별 스코어 테이블 추가
2. 18홀 입력 UI 추가
3. 스트로크 합산 자동 계산
4. 스테이블포드 자동 계산
5. 매치 플레이 승점 계산
6. 신페리오 숨김홀 계산

### 4순위: 관리자 대시보드 요약 강화

운영자가 첫 화면에서 바로 상태를 파악할 수 있도록 아래 요약을 추가한다.

- 예정 라운드 수
- 완료 라운드 수
- 취소 라운드 수
- 최근 생성 라운드
- 최근 스코어 입력 라운드
- 최근 관리자 작업 로그

## 다음 작업 전 체크리스트

다음 기능을 시작하기 전에 아래를 확인한다.

- `npm run verify` 통과 여부
- Supabase SQL 적용 여부
- `/admin` 접속 가능 여부
- `/admin/rounds` 접속 가능 여부
- `/admin/logs` 접속 가능 여부
- 라운드 복제 기능 정상 여부

## 개발 운영 원칙

- 사용자가 요청한 대로 코드와 스크립트 본문은 대화창에 펼치지 않는다.
- 실행 파일 링크와 실행 방법만 제공한다.
- 작업 보고서는 한글로 작성한다.
- 변경 파일은 `dev` 폴더에 기록한다.
- DB 변경이 있으면 Supabase SQL 실행 여부를 반드시 안내한다.
- 큰 기능은 작은 단계로 나누어 진행한다.

## 진행 내용

운영진 회원 관리 화면에 회원 검색 기능을 추가했다.

## 변경 파일

```txt
src/app/(app)/admin/members/page.tsx
dev/2026-06-13-admin-member-search-report.md
```

## 구현 기능

관리자 회원 관리 페이지에서 아래 기준으로 회원을 검색할 수 있다.

```txt
이름
연락처
운영진 / 관리자
회원
연결 완료
연결 대기
미연결
```

검색은 URL query string `q`를 사용한다.

예시:

```txt
/admin/members?q=홍길동
/admin/members?q=0102222
/admin/members?q=연결완료
```

## 구현 방식

이번 단계에서는 DB 구조를 변경하지 않았다.

Supabase에서 같은 동호회 회원 목록을 가져온 뒤, 서버 컴포넌트에서 검색어를 기준으로 필터링한다.

현재 동호회 규모가 작거나 중간 규모인 단계에서는 이 방식이 단순하고 안정적이다.

회원 수가 크게 늘어나면 Supabase 쿼리 조건 기반 검색 또는 전문 검색으로 변경할 수 있다.

## 보안 평가

이번 검색 기능은 관리자 화면 안에서만 동작한다.

관리자 접근 제어는 기존 `requireAdmin()`으로 유지된다.

검색어는 DB raw SQL로 직접 전달하지 않고 서버 컴포넌트 내부 문자열 비교에만 사용하므로 SQL injection 위험이 없다.

# ParkBuddy 조 편성 경기 방식/스마트 편성 통합 개선

## 목적

라운드 목록과 조 편성 화면에서 경기 방식 정보를 한글 중심으로 표시하고, 운영자가 선택한 경기 형태에 따라 조 편성 초안을 더 균등하게 만들기 위한 개선이다.

## 변경 범위

- 라운드 목록의 `점수 계산` 라벨을 `경기 방식`으로 변경
- 경기 형태와 점수 계산 방식을 함께 표시
  - 예: `개인전 신페리오`
- 조 편성 화면의 경기 방식 카드 정리
  - 경기 형태 콤보박스
  - 점수 계산 콤보박스
  - 선택 조합에 맞는 설명 1개만 표시
- `수동 조 편성` 문구를 `조 편성`으로 변경
- 회원 조 선택 카드를 모바일 3열, 태블릿 6열, 데스크톱 9열 compact grid로 변경
- 자동 편성 버튼 추가
- 평균타수와 참여횟수 기반 스마트 조 편성 초안 적용

## 스마트 조 편성 기준

### 공통 실력 점수

- `round_scores.strokes` 평균타수가 낮을수록 높은 실력 점수
- 점수 기록 참여횟수가 많을수록 보정 점수 추가
- 점수 기록이 없는 회원은 핸디캡 기반 기본 점수로 편성

### 개인전 / 스트로크 계열

- 참가자를 실력 점수 높은 순으로 정렬
- 현재 실력 합계가 낮은 조부터 배정
- 특정 조에 고수 또는 신규 회원이 몰리지 않도록 균등 분산

### 포섬 / 포볼 / 청백전

- `round_participants.team_no`에 2개 이상의 팀 값이 있으면 팀 분산을 우선 고려
- 팀 데이터가 부족하면 실력 균등 편성으로 자동 대체
- 현재 ParkBuddy 기본 회원 테이블에는 별도 소속/팀 필드가 없으므로, 팀 기반 2:2 강제 편성은 `team_no` 데이터가 있을 때만 동작한다.

## 화면 문구 원칙

- `ParkBuddy` 브랜드명을 제외하고 사용자 화면에는 한글 문구를 우선 사용한다.
- 조 편성 화면의 경기 방식 카드에서는 콤보박스가 이미 선택값을 보여주므로 `개인전 신페리오` 같은 중복 텍스트는 표시하지 않는다.
- 설명은 선택된 조합을 이해하는 짧은 문장 1개만 표시한다.
- `저장 시 함께 반영` 문구는 표시하지 않는다.

## 확인할 화면

- `/admin/rounds`
- `/admin/rounds/[id]/pairings`

## 확인 포인트

- 라운드 목록에서 `경기 방식` 라벨 아래 `개인전 신페리오`처럼 보이는지
- 조 편성 화면의 경기 방식 카드가 `경기 형태`, `점수 계산`, 설명 1개로 보이는지
- 경기 형태 변경 시 자동 편성이 새 기준으로 다시 적용되는지
- 자동 편성 후 회원별 조 선택 콤보박스로 임의 변경 가능한지
- 저장 후 다시 들어왔을 때 조 편성이 유지되는지
- 회원 카드가 모바일 3열, 태블릿 6열, 데스크톱 9열로 compact하게 보이는지

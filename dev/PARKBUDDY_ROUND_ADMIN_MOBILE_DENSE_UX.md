# PARKBUDDY_ROUND_ADMIN_MOBILE_DENSE_UX

## 작업 목적

라운딩 관리 목록(`/admin/rounds`)은 운영자가 모바일에서 자주 확인하는 화면이므로, 한 화면에서 더 많은 라운딩 정보를 보고 주요 작업으로 빠르게 이동할 수 있도록 compact/dense 레이아웃으로 정리한다.

## 변경 범위

- `src/app/(app)/admin/rounds/page.tsx`

## 변경 내용

### 1. 모바일 헤더 compact 카드화

- 모바일에서 헤더 영역을 흰색 카드로 정리했다.
- 설명 문구를 짧게 줄여 첫 화면 점유 높이를 낮췄다.
- 태블릿 이상에서는 기존처럼 가벼운 헤더로 보인다.

### 2. 상태 필터 sticky compact 적용

- 전체/예정/완료/취소 필터를 모바일 상단 sticky 영역으로 정리했다.
- 카운트 숫자와 라벨 크기를 줄이고 터치 높이는 유지했다.
- 라운딩 목록을 스크롤해도 현재 필터 접근성이 유지된다.

### 3. 라운딩 카드 정보 밀도 개선

- 카드 padding과 gap을 줄였다.
- 라운딩 제목, 상태, 참가자 수를 더 명확한 위계로 정리했다.
- 골프장/날짜/시작 시간/점수 계산 정보는 모바일 2열, 태블릿 이상 4열로 compact하게 배치했다.

### 4. 주요 작업 버튼 4개 즉시 노출

- 참가자, 조 편성, 스코어, 결과 버튼을 모바일에서 4열로 배치했다.
- 터치 높이는 `min-h-11`로 유지해 조작성을 확보했다.
- 상태 관리와 위험 작업은 기존처럼 details 안에 유지했다.

## 검증 포인트

1. `/admin/rounds` 접속
2. 모바일 폭에서 상태 필터가 상단에 붙어 있는지 확인
3. 라운딩 카드가 이전보다 세로로 compact해졌는지 확인
4. 참가자/조 편성/스코어/결과 버튼이 한 줄에 표시되는지 확인
5. 수동 생성 라운딩과 일정 연결 라운딩 모두 정상 표시되는지 확인
6. `npm run verify` 통과 확인

## 주의사항

- 데이터 조회 로직은 변경하지 않았다.
- 라운딩 생성/수정/삭제 서버 액션은 변경하지 않았다.
- 일정 연결 정보 조회 로직은 변경하지 않았다.

# ParkBuddy Event To Round Create Review

## 목적

일정 투표에서 라운딩을 생성할 때 운영자가 생성 결과를 한 번 더 확인할 수 있도록 모바일 친화적인 리뷰 UX를 추가한다.

## 변경 요약

- `/schedule` 운영진 카드의 `참석자 기준 라운딩 생성` 버튼을 접이식 확인 영역으로 변경했다.
- 참석/불참/미선택 인원을 생성 전 한눈에 볼 수 있게 했다.
- 참석 회원만 라운딩 참가자로 추가되고, 불참/미선택 회원은 제외된다는 안내를 추가했다.
- 참석자가 0명인 일정은 버튼 대신 `참석자 필요` 안내 카드만 표시한다.
- 이미 라운딩이 생성된 일정은 기존처럼 `생성된 라운딩 보기` CTA를 유지한다.

## 운영자 확인 포인트

1. `/schedule`에서 참석자가 있는 일정의 `참석자 기준 라운딩 생성` 영역을 펼친다.
2. 참석/불참/미선택 수가 투표 현황과 맞는지 확인한다.
3. `확인 후 라운딩 생성`을 눌러 라운딩이 생성되는지 확인한다.
4. 생성 후 같은 일정 카드가 `생성된 라운딩 보기`로 바뀌는지 확인한다.
5. 참석자가 없는 일정은 생성 버튼이 노출되지 않는지 확인한다.

## 주의

이 작업은 UI 안전장치만 추가한다. 중복 생성 방지는 기존 `admin_create_round_from_event` RPC와 `rounds.event_id` 유니크/검증 흐름을 계속 신뢰한다.

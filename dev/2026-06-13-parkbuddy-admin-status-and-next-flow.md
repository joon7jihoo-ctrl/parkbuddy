# ParkBuddy 운영자용 개발 현황 및 점검 문서

작성일: 2026-06-13
작성 목적: 지금까지 구현된 기능을 정리하고, 다음 개발을 안정적으로 이어가기 위한 기준 문서

---

## 1. 현재 개발 원칙

- 앱 코드를 변경하기 전에는 목적, 영향 파일, 검증 방법을 먼저 정리한다.
- 실행 파일은 기본적으로 `.ps1` 하나로 제공한다.
- 코드와 스크립트 본문은 대화창에 펼치지 않는다.
- 작업 보고서와 운영 로그는 한글 기준으로 작성한다.
- 새 기능 추가보다 안정화가 우선이다.
- Supabase 스키마를 추측하지 않고, 실제 스키마 확인 후 SQL/RPC를 작성한다.
- 앱 코드 변경 후에는 `npm run verify`를 통과해야 다음 단계로 넘어간다.
- Supabase SQL이 필요한 작업은 반드시 SQL Editor 실행 여부를 별도로 확인한다.

## 3. 최근 안정화 이슈

### 반복된 스크립트 파싱 오류

원인:
- PowerShell에 긴 TSX/SQL/한글 문자열을 직접 삽입하면서 문자열 종료/인코딩 문제가 반복 발생했다.

현재 기준:
- 앞으로는 `.ps1` 파일을 사용하되, 내부 문자열은 가능한 Base64 또는 아주 단순한 파일 조작 방식으로 처리한다.
- 한글 콘솔 출력은 최소화한다.
- 스크립트 안에서 복잡한 한글 보고서 배열 생성은 피한다.
- 보고서 파일은 가능하면 UTF-8 파일 쓰기 방식으로 생성한다.

### 대시보드 UI 변경 이슈

원인:
- 최근 관리자 작업 한글화 과정에서 `/admin/page.tsx` 전체가 의도보다 많이 변경됐다.

현재 조치:
- 대시보드 안정 UI 복구 파일을 적용했다.
- 다음 대시보드 수정 전 반드시 스크린샷 저장 후 진행한다.
- 앞으로 대시보드 수정은 전체 파일 덮어쓰기보다 작은 변경 단위로 진행한다.

### 라운드 복제 이슈

발생했던 문제:
- `rounds.deleted_at` 컬럼이 없는데 참조
- `members.auth_user_id` 컬럼이 없는데 참조
- `admin_log_action` 함수 인자 수 불일치
- 라운드 복제 시 날짜/시간이 복제 시점 기준으로 들어가지 않음

현재 기준:
- 실제 Supabase 스키마 점검 CSV 기준으로 수정했다.
- `members.user_id`는 존재한다.
- `rounds.deleted_at`은 존재하지 않는다.
- `admin_log_action`은 실제 함수 시그니처 기준으로 호출해야 한다.
- 복제 시 경기 형태/점수 계산 방식은 유지하고, 날짜/시작시간은 복제 시점 기준으로 생성한다.

## 4. Supabase SQL 실행이 필요한 기능

아래 기능은 로컬 파일 변경만으로 끝나지 않고 Supabase SQL Editor 실행이 필요하다.

- 라운드 기본 테이블/RPC 관련 마이그레이션
- 라운드 참가자 테이블/RPC
- 조 편성 테이블/RPC
- 스코어 테이블/RPC
- 라운드 상태 변경 RPC
- 라운드 복제 RPC
- 라운드 복제 날짜/시간 현재값 보정 RPC

다음 개발 전 확인:
- `supabase/migrations` 폴더의 최신 SQL이 Supabase SQL Editor에 실제로 반영됐는지 확인한다.

## 5. 다음 개발 전 필수 체크리스트

### 로컬 앱 상태

- `npm run verify` 통과
- `/admin` 대시보드 UI 확인
- `/admin/rounds` 라운드 목록 확인
- 라운드 생성/수정/복제 버튼 확인
- 라운드 복제 시 날짜/시간 확인
- `/admin/logs` 작업 로그 한글 표시 확인

### Supabase 상태

- 최신 migration SQL 실행 여부 확인
- `admin_duplicate_round` 함수 최신 정의 반영 확인
- `admin_log_action` 함수 시그니처 확인
- 최근 작업 로그에 필요한 action들이 기록되는지 확인

### Git 상태

- `.env.local` 등 비밀 정보가 커밋되지 않는지 확인
- `node_modules`, `.next`, `.env*` 제외 확인
- `dev` 문서는 커밋 대상에 포함
- 백업 `.bak` 파일은 필요 여부 확인 후 커밋 제외 권장

## 6. 다음 추천 개발 순서

### 1순위: Git 저장소 정리 및 원격 백업

현재 기능이 많이 쌓였기 때문에 다음 기능 추가 전 Git에 올리는 것이 최우선이다.

### 2순위: 대시보드 UI 안정화

현재 원하는 UI 기준으로 스크린샷을 남기고, 이후 대시보드 변경은 작은 패치만 허용한다.

### 3순위: 라운드 복제 최종 확인

- 복제 후 날짜/시작시간이 현재 기준인지 확인
- 경기 형태/점수 계산 방식 유지 확인
- 참가자/조 편성/스코어 미복사 확인
- 관리자 로그 기록 확인

### 4순위: 스코어 입력 UX 개선

- 스코어 미입력자 강조
- 저장 후 결과 보기 바로가기
- 입력값 검증 메시지 개선

### 5순위: 홀별 스코어 입력

신페리오/매치 플레이를 제대로 계산하려면 18홀 단위 데이터가 필요하다. 다음 대형 기능은 홀별 스코어 입력 기반을 먼저 설계한 뒤 진행한다.

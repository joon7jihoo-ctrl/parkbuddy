# ParkBuddy 라운드 목록 Dense UX 및 조 편성 자동 배정 복구

## 목적

라운딩 관리 목록의 모바일 Dense UX가 이후 패치에서 원복된 문제를 다시 바로잡고, 조 편성 화면에서 기존 저장값이 모두 1조로 남아 있을 때 자동 조 편성이 적용되지 않는 문제를 방어한다.

## 변경 내용

### 1. `/admin/rounds` 모바일 Dense UX 재적용

- 모바일 헤더를 compact 카드형으로 다시 정리했다.
- 전체/예정/완료/취소 상태 필터를 모바일 sticky 영역으로 재적용했다.
- 라운드 카드는 compact padding/gap을 사용한다.
- 골프장/날짜/시작 시간/경기 방식 정보를 모바일 2열, 태블릿 이상 4열로 표시한다.
- 참가자/조 편성/스코어/결과 버튼은 모바일에서 4열로 즉시 접근 가능하게 복구했다.
- `경기 방식` 라벨 아래에는 `개인전 신페리오`처럼 경기 형태와 점수 계산 방식을 함께 표시한다.

### 2. 조 편성 자동 배정 보정

- 기존 저장 조 편성값이 불완전하거나 특정 조에 과도하게 몰린 경우에는 저장값을 그대로 신뢰하지 않고 스마트 자동 편성을 적용한다.
- 예: 참가자가 8명인데 기존 저장값이 모두 1조인 경우, 한 조 8명은 유효하지 않으므로 자동 편성으로 재분산한다.
- 경기 형태를 바꾸면 해당 경기 형태 기준으로 자동 편성을 다시 적용한다.
- 점수 계산 방식을 바꿔도 화면의 조 편성 초안이 최신 기준으로 다시 계산된다.
- 참가 인원이 3~4명이라 실제로 한 조만 가능한 경우에는 `현재 참가 인원은 한 조 편성이 적절합니다.`라고 안내한다.

## 확인 항목

1. `/admin/rounds` 모바일 화면에서 상태 필터가 sticky compact 영역으로 보이는지 확인한다.
2. 라운드 카드의 버튼이 모바일에서 참가자/조 편성/스코어/결과 4열로 보이는지 확인한다.
3. 라운드 목록의 `경기 방식` 아래에 경기 형태와 점수 계산 방식이 함께 보이는지 확인한다.
4. 참가자 5명 이상 라운드의 조 편성 화면에서 모든 인원이 1조로 몰리지 않는지 확인한다.
5. 경기 형태 변경 시 조 편성이 다시 자동 재분산되는지 확인한다.
6. 참가자가 3~4명인 라운드는 한 조만 표시되는 것이 정상인지 확인한다.

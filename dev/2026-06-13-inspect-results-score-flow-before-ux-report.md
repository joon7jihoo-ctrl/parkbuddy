# 결과 화면 UX 개선 전 안전 점검

## 목적

결과 화면에서 미입력 회원을 스코어 입력 화면으로 자연스럽게 연결하기 전에, 현재 결과/스코어 관련 파일 구조를 먼저 확인한다.

## 점검 대상

- src/app/(app)/admin/rounds/[id]/results/page.tsx
- src/app/(app)/admin/rounds/[id]/scores/page.tsx
- src/app/(app)/admin/rounds/[id]/scores/actions.ts

## 원칙

이번 단계에서는 앱 코드를 수정하지 않는다.

다음 구현 단계에서는 다음 항목을 최소 수정으로 진행한다.

1. 결과 화면에서 미입력 인원이 있을 때 스코어 입력 버튼을 더 눈에 띄게 표시
2. 입력 완료/미입력 상태를 결과 화면에서 명확히 표시
3. 기존 결과 정렬, 인쇄, 링크 복사 기능 유지
4. DB 변경 없이 화면 UX만 개선

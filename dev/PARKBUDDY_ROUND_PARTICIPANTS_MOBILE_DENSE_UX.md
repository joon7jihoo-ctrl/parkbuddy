# ParkBuddy 라운딩 참가자 관리 모바일 Dense UX

## 목적

라운딩 목록 화면 compact 개선 이후, 운영자가 가장 자주 사용하는 참가자 관리 화면(`/admin/rounds/[id]/participants`)도 모바일에서 더 적은 스크롤로 조작할 수 있도록 정리한다.

## 변경 범위

- `src/app/(app)/admin/rounds/[id]/participants/page.tsx`
- `src/components/admin/participant-selection-enhancer.tsx`

## 적용 내용

### 참가자 관리 화면

- 페이지 여백과 섹션 간격을 줄여 모바일 밀도를 높였다.
- 라운드 제목/코스/일시 영역을 흰색 compact 헤더 카드로 정리했다.
- 모바일 요약 카드 padding을 줄였다.
- 참가자 검색 UI를 form 내부에 배치해 저장 흐름과 가깝게 만들었다.
- 회원 카드 높이, padding, 글자 크기를 줄였다.
- 연락처/핸디캡 표기를 짧게 정리했다.
- 체크박스 크기를 28px로 키워 모바일 터치성을 유지했다.
- 저장 버튼의 선택 인원 숫자가 체크박스 변경에 따라 함께 갱신되도록 data attribute를 추가했다.

### 참가자 검색 도구

- 검색/선택 필터 영역을 모바일 sticky compact card로 변경했다.
- 검색 input 높이를 48px로 줄였다.
- `선택만 보기`, `표시 전체 선택` 버튼을 2열로 배치했다.
- 선택 인원 요약 chip을 우측 상단에 추가했다.
- 선택 인원 변경 시 화면 내 선택 count output을 함께 갱신한다.

## 검증 포인트

- `/admin/rounds/[id]/participants` 접속 시 화면이 정상 렌더링되는지 확인한다.
- 모바일 폭에서 검색 카드가 상단에 compact하게 유지되는지 확인한다.
- 회원 검색이 이름/연락처/역할 기준으로 동작하는지 확인한다.
- 선택만 보기/전체 보기 토글이 정상 동작하는지 확인한다.
- 표시 전체 선택/표시 해제가 보이는 항목 기준으로 동작하는지 확인한다.
- 체크박스 변경 시 상단 검색 카드, 안내 문구, 저장 버튼의 선택 수가 함께 갱신되는지 확인한다.
- 참가자 저장 후 `updated=1` 성공 안내가 정상 표시되는지 확인한다.

## 주의

- 참가자 저장 서버 액션과 Supabase RPC는 변경하지 않았다.
- 라운딩 목록, 조 편성, 스코어 입력, 결과 화면은 이번 패치에서 변경하지 않았다.

# ParkBuddy 라운드 관리 공통 레이아웃 표준

## 작업명

Apply consistent round management layouts

## 목적

`/admin/rounds`에서 정리한 모바일 우선 카드 레이아웃을 라운드 관리 계열 화면 전체에 확장 적용한다.

## 적용 대상

- `/admin/rounds`
- `/admin/rounds/calendar`
- `/admin/rounds/status`
- `/admin/rounds/deleted`

## 공통 레이아웃 원칙

1. 모바일에서 먼저 보기 좋은 구조를 기준으로 한다.
2. 핵심 정보는 2열 카드형 정보 그리드로 배치해 세로 스크롤을 줄인다.
3. 데스크탑에서는 `max-w-7xl`을 사용해 라운드 내용이 왼쪽에 좁게 눌리지 않도록 한다.
4. 라운드 제목, 상태, 참가자 수 같은 핵심 정보는 카드 상단에 항상 노출한다.
5. 참가자, 조 편성, 스코어, 결과 같은 핵심 작업 버튼은 먼저 보이게 한다.
6. 메모와 위험 작업은 필요할 때만 펼칠 수 있게 한다.
7. 삭제된 라운드 화면은 복구 중심으로 단순화하고 일반 운영 버튼은 표시하지 않는다.
8. 월별 일정/상태별 보기 화면도 라운드 목록과 같은 버튼, 카드, 정보 그리드 스타일을 사용한다.

## 검증 기준

- `npm run verify` 통과
- 데스크탑에서 라운드 정보가 좁게 눌리지 않음
- 모바일에서 기본 정보가 한 화면에 더 많이 들어옴
- `/admin/rounds/calendar`와 `/admin/rounds/status`가 `/admin/rounds`와 같은 카드 패턴으로 보임
- `/admin/rounds/deleted`는 복구 중심 UI만 제공함
- 삭제된 라운드는 월별/상태별 일반 목록에 노출되지 않음

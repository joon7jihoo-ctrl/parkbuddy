# ParkBuddy 일정 투표 상세 명단 UX 개선

## 작업 일자
- 2026-06-16

## 작업 범위
- `/schedule` 일정 카드 투표 UI
- 참석/불참 투표 결과 표시
- 투표자 명단 팝업

## 변경 파일
- `src/app/(app)/schedule/page.tsx`
- `src/app/(app)/schedule/VoteButtons.tsx`
- `dev/PARKBUDDY_SCHEDULE_POLL_DETAIL_MODAL.md`
- `dev/PARKBUDDY_OPERATOR_DEV_STATUS.md`

## 반영 내용
1. `참석 여부` 문구 삭제
2. 기존 `내 선택` 텍스트 삭제
3. 체크 표시 UI를 라디오 버튼 형태로 변경
4. 선택 색상을 노란색에서 ParkBuddy 프로젝트 컬러 계열인 emerald/brand 계열로 변경
5. 참석/불참 각 항목에 전체 활성 회원 대비 선택 비율을 가로 막대그래프로 표시
6. 참석/불참 선택 시 즉시 내 선택을 반영하고 해당 선택 명단 팝업 표시
7. 일정 유형 배지 아래에 전체 투표 인원수와 비율 표시
8. 전체 투표 인원 버튼 선택 시 참석/불참 전체 명단 팝업 표시
9. 기존 `maybe` 값은 투표 선택지에서는 제외하고, 참석/불참 투표 카운트와 명단에서도 제외

## UX 기준
- 모바일 Dense Layout 유지
- 투표 버튼 터치 영역 44px 이상 유지
- 팝업은 모바일에서 하단 시트처럼 표시
- 데스크탑에서는 중앙 모달처럼 표시
- 투표 결과 색상은 프로젝트 기본 색상 계열을 사용

## 검증
- `npm run verify` 통과
  - security smoke test 통과
  - eslint 통과
  - typecheck 통과

## 화면 확인 체크리스트
1. `/schedule` 접속
2. 일정 유형 배지가 `정기 라운딩 / 대회 / 번개`로 표시되는지 확인
3. 배지 아래 `투표 n명 · n%` 버튼이 표시되는지 확인
4. `투표 n명` 버튼 선택 시 전체 참석/불참 명단 팝업이 뜨는지 확인
5. 참석 라디오 항목 선택 시 내 선택이 즉시 반영되고 참석 명단 팝업이 뜨는지 확인
6. 불참 라디오 항목 선택 시 내 선택이 즉시 반영되고 불참 명단 팝업이 뜨는지 확인
7. 참석/불참 항목 옆 비율과 가로 막대가 전체 활성 회원 대비 비율로 보이는지 확인
8. 모바일/태블릿/데스크탑에서 버튼과 팝업이 자연스럽게 보이는지 확인
